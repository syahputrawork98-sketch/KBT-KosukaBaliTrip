# Project History

Folder ini berisi riwayat pengerjaan project Kosuka Bali Trip.

Dokumentasi ini dibuat agar status project, batch terakhir, commit terakhir, dan langkah berikutnya tetap mudah dipahami walaupun akun ChatGPT, room chat, atau AI tools yang digunakan berubah.

## Tujuan

Project history digunakan untuk mencatat:
- Batch pekerjaan yang sudah selesai
- Batch yang sedang berjalan
- Commit hash penting
- Hasil pekerjaan
- Catatan review
- Langkah berikutnya

## Sistem Batch

Project ini dikerjakan secara bertahap menggunakan sistem batch.

Satu file history digunakan untuk merangkum 10 batch.

Contoh:
- `BATCH_000_TO_010.md`
- `BATCH_011_TO_020.md`
- `BATCH_021_TO_030.md`

Dengan sistem ini, dokumentasi tetap rapi dan tidak terlalu banyak file.

## File Utama

### CURRENT_STATUS.md

File ini berisi status project terbaru.

Baca file ini terlebih dahulu untuk mengetahui:
- Project terakhir sampai mana
- Batch terakhir yang selesai
- Batch yang sedang berjalan
- Commit terakhir
- Langkah berikutnya

### BATCH_000_TO_010.md

File ini berisi catatan Batch 000 sampai Batch 010.

### BATCH_TEMPLATE.md

File ini berisi format standar untuk mencatat batch di dalam file history.

## Alur Membaca untuk AI Baru

Jika akun ChatGPT, room chat, atau AI tools berubah, baca dokumen dengan urutan berikut:

1. `README.md` di root repository
2. `FEATURES.md` di root repository
3. `docs/README.md`
4. `docs/project-workflow/README.md`
5. `docs/project-workflow/PROJECT_CONTEXT.md`
6. `docs/project-workflow/WORKING_SYSTEM.md`
7. `docs/project-history/CURRENT_STATUS.md`
8. File batch history yang relevan, misalnya `docs/project-history/BATCH_000_TO_010.md`

## Prinsip Utama

GitHub adalah Source of Truth utama.

Project history hanya valid jika sesuai dengan isi repository dan commit hash yang tercatat.
