# Project Roadmap

Roadmap ini membagi siklus pengembangan project Kosuka Bali Trip ke dalam beberapa fase utama.

## 1. Phase Planning
- Pengumpulan requirement (kebutuhan fitur).
- Pembagian scope kerja menjadi batch-batch pengembangan.
- Menentukan batasan (MVP vs Non-MVP).

## 2. Phase Documentation
- Pembuatan dokumen fondasi (README, Brief, Roadmap, MVP Scope).
- Penyusunan dokumentasi teknis, database, dan skema API.

## 3. Phase Development MVP
- Setup framework dan arsitektur aplikasi (frontend & backend).
- Pembuatan UI/UX komponen.
- Integrasi frontend dengan backend API.

## 4. Phase Testing
- Quality Assurance (QA) testing secara manual.
- User Acceptance Testing (UAT) untuk fungsionalitas inti (seperti flow redirect WA).

## 5. Phase Deployment
- Setup hosting/VPS dan domain.
- Rilis Minimum Viable Product (MVP) ke production server.

## 6. Phase Improvement
- Evaluasi feedback dari user/admin.
- Pengembangan fitur tambahan yang tertunda (Payment gateway, CMS Admin, dll).
