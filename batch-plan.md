# Batch Plan

Rencana eksekusi proyek dibagi menjadi 10 Batch kerja yang dilakukan secara bertahap.

| Batch | Nama Batch | Fokus Pengerjaan | Status |
| :---: | :--- | :--- | :---: |
| **Batch 01** | **Project Documentation Foundation** | Membuat struktur folder `docs/`, `README.md`, dan fondasi dokumen (brief, roadmap, mvp, batch plan, changelog). Belum ada kode yang ditulis. | ✅ **Done** |
| **Batch 02** | **Technical Setup & Architecture** | Inisialisasi package manager, setup linter, formatter, environment variables, dan standar arsitektur (Tanpa implementasi fitur bisnis). | ⏳ Planned |
| **Batch 03** | **Database & Backend Foundation** | Merancang ERD, setup koneksi database, skema tabel, dan struktur dasar backend. | ⏳ Planned |
| **Batch 04** | **API Development - Core** | Membuat endpoint API utama (GET paket wisata, POST inquiries, dsb) untuk melayani frontend. | ⏳ Planned |
| **Batch 05** | **Frontend Foundation & Routing** | Setup framework frontend, konfigurasi global state, styling setup (Tailwind/CSS), dan inisiasi routing dasar. | ⏳ Planned |
| **Batch 06** | **Frontend - UI Components & Pages** | Slicing UI dari desain ke komponen kode (Hero, Cards, Footer, dll) dan merakit halaman (Home, Detail Paket). | ⏳ Planned |
| **Batch 07** | **Backend & Frontend Integration** | Menghubungkan fungsionalitas UI dengan data nyata dari API Backend (Fetching data paket wisata, validasi form booking). | ⏳ Planned |
| **Batch 08** | **Admin Panel / CMS Setup** | (Opsional untuk MVP/Sederhana) Pembuatan dashboard admin basic untuk mengelola data katalog wisata. | ⏳ Planned |
| **Batch 09** | **Testing & Bug Fixing** | Melakukan flow testing end-to-end, memastikan form redirect WA berfungsi sempurna, dan perbaikan UI bugs. | ⏳ Planned |
| **Batch 10** | **Deployment & Launch** | Konfigurasi CI/CD, setup environment production, deploy ke cloud/hosting, dan go-live. | ⏳ Planned |
